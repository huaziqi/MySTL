### POD类型

POD的全拼是plain old data，指的是int，char这种单纯的原始数据。string类型的数据保存在堆上，string本身只保存指针，所以有析构函数，需要单独析构，复制时也需要单独处理

### static

| 位置           | static 的效果          |
| -------------- | ---------------------- |
| 全局变量/函数  | 限制为文件内可见       |
| 函数内局部变量 | 延长生命周期到程序结束 |
| 类成员         | 属于类而非实例         |

### 为什么 `_true_type`/`_false_type` 要定义为结构体

目的是在**编译期**通过函数重载选择实现，而非运行期 `if` 判断：

```cpp
struct _true_type {};
struct _false_type {};

template<class T>
void copy_impl(T* dst, T* src, _true_type) {   // POD：直接 memcpy
    memcpy(dst, src);
}
template<class T>
void copy_impl(T* dst, T* src, _false_type) {  // 非POD：逐个拷贝构造
    // ...
}
template<class T>
void copy(T* dst, T* src) {
    copy_impl(dst, src, typename _type_traits<T>::is_POD_type{});
}

```

|          | `bool` 值      | `_true_type`/`_false_type` |
| -------- | -------------- | -------------------------- |
| 判断时机 | 运行期         | 编译期                     |
| 选择机制 | `if/else`      | 函数重载                   |
| 性能     | 两个分支都编译 | 只编译匹配的那个           |

### `if/else` 在编译期会导致性能下降？

不完全准确，需要区分两种情况。

**普通 `if/else`（运行期）：**

```cpp
void copy(T* dst, T* src) {
    if (_type_traits<T>::is_POD)  // 这是个 bool 值
        memcpy(dst, src);
    else
        // 逐个拷贝构造...
}
```

条件是编译期常量时，现代编译器开优化后会**直接消除死分支**，实际性能损失几乎为零。但问题是**两个分支都必须能通过编译**，如果 `else` 分支里有对非 POD 类型才合法的操作，即使永远不会执行，编译器也会报错。

**`if constexpr`（C++17，真正的编译期分支）：**

```cpp
if constexpr (_type_traits<T>::is_POD) {
    memcpy(dst, src);
} else {
    // 逐个拷贝构造...  ← 对 POD 类型时，这段代码直接不编译
}
```

不满足条件的分支**完全不参与编译**，解决的是编译问题而非性能问题。

**结论：** 用 `_true_type`/`_false_type` 的真正原因是 C++11 之前没有 `if constexpr`，函数重载是唯一能做到"让不匹配的分支不参与编译"的手段。性能是次要的，**编译期类型安全**才是主要动机。

### 偏特化写法

模板参数分两种：

```cpp
template<bool B, class Ta, class Tb>
//       ^^^^^^  ^^^^^^^^  ^^^^^^^^
//       值参数   类型参数   类型参数
```

- **`class Ta`** — 接收一个类型，如 `int`、`double`
- **`bool B`** — 接收一个编译期的值，如 `true`、`false`，称为**非类型模板参数**

偏特化规则：**至少固定一个参数**，剩余参数在 `template<>` 里声明为自由变量，特化尖括号里直接使用变量名，不加 `class`：

```cpp
// 主模板
template<bool B, class Ta, class Tb>
struct IfThenElse;

// 偏特化：固定 bool = true，Ta/Tb 仍待定
template<class Ta, class Tb>
struct IfThenElse<true, Ta, Tb> {
    using result = Ta;
};

// 偏特化：固定 bool = false，Ta/Tb 仍待定
template<class Ta, class Tb>
struct IfThenElse<false, Ta, Tb> {
    using result = Tb;
};
```

固定参数的选择决定了特化的语义：

| 固定的参数 | 含义 |
|---|---|
| `bool = true` | 条件为真时的行为 |
| `bool = false` | 条件为假时的行为 |
| `Ta = int` | Ta 恰好是 int 时的特殊处理 |
| `bool = true, Ta = int` | 条件为真且 Ta 是 int 时的特殊处理 |

常见的非类型模板参数：

```cpp
template<int N>     // 整数，如 std::array<int, 5> 里的 5
template<bool B>    // bool
template<size_t N>  // 常用于数组大小
```

### Construct.h — 对象构造与析构

STL 把对象生命周期拆成四个独立操作：

```
allocate()    — 分配原始内存
construct()   — 在内存上构造对象
destroy()     — 析构对象
deallocate()  — 释放内存
```

**`construct` — placement new**

```cpp
template<class T1, class T2>
inline void construct(T1* ptr, const T2& value) {
    new(ptr) T1(value);
}
```

`new(ptr) T(value)` 是 placement new 语法，需要 `#include <new>`：

```cpp
new(ptr) T(value)
// ^^^         — 目标地址，不分配内存
//      ^      — 要构造的类型
//        ^^^  — 传给构造函数的参数
```

对比普通 `new`：

```cpp
T* p = new T(value);      // 分配内存 + 构造
       new(ptr) T(value); // 只构造，内存已经有了
```

`T1` 和 `T2` 是两个类型参数，允许参数类型和目标类型不完全一致，只要能隐式转换即可：

```cpp
construct(ptr, 3.14);  // T1=int，T2=double
construct(ptr, "hi");  // T1=string，T2=char*
```

---

**`destroy` — 析构**

```cpp
// 单个对象
template<class T>
inline void destroy(T* ptr) {
    ptr->~T();
}

// 范围析构，POD 直接跳过，非 POD 逐个调析构
template<class ForwardIterator>
inline void destroy(ForwardIterator first, ForwardIterator last) {
    typedef typename _type_traits<ForwardIterator>::is_POD_type is_POD_type;
    _destroy(first, last, is_POD_type());
}
```

- `destroy` 只调析构函数，**不释放内存**，释放内存是 `deallocate` 的工作
- POD 类型也可以显式调用析构，编译器生成空的伪析构函数，什么都不做
- 范围版本需要区分 POD/非 POD：对一百万个 `int` 逐个调空析构，循环本身有开销，POD 直接跳过整个循环

**析构函数的本质：** 释放对象内部自己申请的资源，是构造的逆操作。

```
string 对象（栈）          堆
┌─────────────┐           ┌───────────────┐
│ ptr ────────┼──────────►│ h e l l o \0  │
│ size = 5    │           └───────────────┘
└─────────────┘
```

`~string()` 里 `delete[] ptr` 释放堆上字符数组，对象本身占的栈空间由调用方负责。POD 类型没有内部资源，所以析构为空。
